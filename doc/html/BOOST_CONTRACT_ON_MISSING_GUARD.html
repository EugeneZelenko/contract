Macro BOOST_CONTRACT_ON_MISSING_GUARD

BOOST_CONTRACT_ON_MISSING_GUARD — Code block to execute if contracts are not assigned to a boost::contract::guard variable (undefined by default).

Synopsis

// In header: <boost/contract/core/config.hpp>

BOOST_CONTRACT_ON_MISSING_GUARD

Description

In general, there is a logic error in the program when contracts are not assigned to a local variable of type boost::contract::guard. Therefore, by default (i.e., when this macro is not defined) this library calls assert with a false condition in such cases. If this macro is defined, this library will execute the code expanded by the macro instead of calling assert (if programmers prefer to throw an exception, etc.).

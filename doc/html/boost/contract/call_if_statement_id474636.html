Struct template call_if_statement<true, Then, unspecified>

boost::contract::call_if_statement<true, Then, unspecified> — Template specialization to dispatch between then-branch functor calls that return void and the ones that return non-void.

Synopsis

// In header: <boost/contract/call_if.hpp>

template<typename Then> 
struct call_if_statement<true, Then, unspecified> : public boost::contract::call_if_statement< true, Then, result_of< Then()>::type >
{
  // construct/copy/destruct
  explicit call_if_statement(Then);
};

Description

The base class is a call-if statement so the else and else-if statements can be specified if needed.

Usually this class template is instantiated only via the return value of boost::contract::call_if and boost::contract::call_if_c.

Note

The result_of<Then()>::type expression should be evaluated only when the static condition is already checked to be true (because Then() is required to compile only in that case). This template specialization introduces an extra level of indirection necessary for the proper lazy evaluation of this result-of expression.

Template Parameters

  1. typename Then

    Type of functor to call when the static condition if true.

call_if_statement public construct/copy/destruct

  1. explicit call_if_statement(Then f);
    Construct this object with the then-branch functor.

    Parameters:

    f

    Then-branch nullary templated functor. The functor f() is compiled and called in this case (because the if-statement static condition is true for this template specialization). The return type of f() must be the same as (or implicitly convertible to) the return type of all other functor calls specified for this call-if object.

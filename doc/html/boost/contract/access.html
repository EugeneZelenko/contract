Class access

boost::contract::access — Friend this class to declare invariants and base types as private members.

Synopsis

// In header: <boost/contract/core/access.hpp>


class access : private noncopyable {
};

Description

Declare this class as friend of the contracted class in order to declare the invariants functions and the base types typedef as non-public members. In real code, programmers will likely chose to declare this class as friend so to fully control the contracted class public interface.

This class is not intended to be directly used by programmers a part from declaring it friend (and that is why this class does not have any public member and it is not copyable).

Note

Not making this class friend of the contracted class will cause compiler errors on some compilers (e.g., MSVC) because the private members needed to check the contracts will not be accessible. On other compilers (e.g., GCC and CLang), the private access will instead fail SFINAE and no compiler error will be reported while invariants and subcontracting will be silently skipped at run-time. Therefore, programmers must make sure to either declare invariant functions and base types typedef as public members or to make this class a friend.

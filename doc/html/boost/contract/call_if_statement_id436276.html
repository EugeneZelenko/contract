Struct template call_if_statement<false, Then, unspecified>

boost::contract::call_if_statement<false, Then, unspecified> — Template specialization to handle false static conditions.

Synopsis

// In header: <boost/contract/call_if.hpp>

template<typename Then> 
struct call_if_statement<false, Then, unspecified> {
  // construct/copy/destruct
  explicit call_if_statement(Then const &);

  // public member functions
  template<typename Else> result_of< Else()>::type else_(Else) const;
  template<bool ElseIfCond, typename ElseIfThen> 
    call_if_statement< ElseIfCond, ElseIfThen > else_if_c(ElseIfThen) const;
  template<typename ElseIfCond, typename ElseIfThen> 
    call_if_statement< ElseIfCond::value, ElseIfThen > 
    else_if(ElseIfThen) const;
};

Description

This one specialization handles all else-branch functor calls (whether they return void or not).

Usually this class template is instantiated only via the return value of boost::contract::call_if and boost::contract::call_if_c.

call_if_statement public construct/copy/destruct

  1. explicit call_if_statement(Then const & f);
    Construct this object with the then-branch functor.

    Parameters:

    f

    Then-branch nullary templated functor. The functor call f() is never compiled and executed in this case (because the if-statement static condition is false for this template specialization). The return type of f() must be the same as (or implicitly convertible to) the return type of the other functor calls specified for this call-if object.

  1. template<typename Else> result_of< Else()>::type else_(Else f) const;
    Specify the else-branch functor.
    Note

    The result_of<Else()>::type expression should be evaluated only when the static condition is already checked to be false (because Else() is required to compile only in that case). Thus, this result-of expression is evaluated lazily only in instantiations of this template.

    Parameters:

    f

    Else-branch nullary templated functor. The functor call f() is compiled and executed in this case (because the if-statement static condition is false for this template specialization). The return type of f() must be the same as (or implicitly convertible to) the return type of the other functor calls specified for this call-if object.

    Returns:

    A copy of the value returned by the call to the else-branch functor f().

    Specify an else-if-branch functor.

    f

The functor call f() is compiled and executed if and only if ElseIfCond is true (because the if-statement static condition is false for this template specialization). The return type of f() must be the same as (or implicitly convertible to) the return type of the other functor calls specified for this call-if object.
